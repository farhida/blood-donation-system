1) Title
  - Blood Donation System
  - Stack: Django (DRF) backend, React frontend

2) Agenda

3) Architecture
  - SPA frontend -> /api -> Django REST backend
  - Supports sqlite (dev), Postgres/MySQL (prod)

4) Key features
  - Registration/login (JWT)
  - Donor search (public)
  - User profile & dashboard
  - Admin pages and admin-only APIs
  - Availability computed from `last_donation` and phone privacy respected

5) How to run (brief)
  - Backend: use virtualenv, pip install -r backend/requirements.txt, migrate, runserver
  - Frontend: cd frontend, npm install, npm start; build with npm run build

6) Deployment notes
  - Use `DATABASE_URL` for production DB
  - Frontend uses `runtime-config.json` at root to set API host at runtime
  - Admin creation can be done in build scripts via env vars

7) Next steps
  - Add smoke tests and CI, tighten CORS & security headers, add cache-busting for frontend bundles
